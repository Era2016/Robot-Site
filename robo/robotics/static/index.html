Bolt CSS Guidelines

Bolt is designed to be a feature rich web application and since there are many people working on the design at the same time, I thought it would be important to put together this guide.

In retrospect it was probably a better idea to start with bootstrap but me being me, I don't like my work looking like bootstrap. Or foundation. Yes, it would have been much easier to implement, so sorry (but not really).

Basics & Grid

This semi-responsive grid system follows a similar convention to most other layouts. It's designed for 1280px screens, which should be sufficient for almost every computer. This grid is semi-responsive, as it adjusts its layout for screens/windows less than 767 pixels. Feel free to expand the CSS to accomodate for more screen sizes!

Container

The outermost wrapper is typically the <div class="container"> ... </div> . This wrapper provides a 1180 total width for the outermost box and contains a 20 pixel padding on the left and the right side. If you skip the container, the columns will fit to the entire width of the window. This should not be common, except for the navigation menu (and this may change in the near future).

Row

Rows encapsulate the child columns to prevent any unexpected wrapping when screens get too tiny to render each column. This setup is typically needed if there are many stages between desktop and mobile, but in our case (for the moment), we only have the two. Rows are called using <div class="row"> ... </div>

Column

Columns are sized by width percentage and spaced using a left margin. The exact size of the column is dependent on it's parent width. If you're not sure, just wrap the columns in a new container. Containers are called using <div class="columns #"> ... </div> and I will get into them further in the next paragraphs.

To recap here is a typical setup: 

        <div class="container">
            <div class="row">
                <div class="columns six">
                    <p>This is a column of text</p>
                </div>
                <div class="columns six">
                    <p>This is a column text too!</p>
                </div>
            </div>
        </div>

Typography

This style's HTML is rendered at a font-size of 62.5%, which allows us to use 1em = 10px. The recommendation is to use em for fonts, as they take the parent font-size as the basis for its size and rem for padding and emergency cases, because they take the root (html) font-size as the basis for its size.

Headlines
